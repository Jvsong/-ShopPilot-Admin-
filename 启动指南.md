# 电商后台管理系统 - 启动指南

## 项目结构
- `backend/` - Spring Boot 后端 (端口: 8083)
- `frontend/` - Vue 3 前端 (端口: 3000)
- `database/` - 数据库脚本和配置

## 当前状态
✅ **后端已在运行** - 端口 8083  
❌ **前端未运行** - 需要启动

## 启动前端

### 1. 进入前端目录
```bash
cd frontend
```

### 2. 安装依赖（如未安装）
```bash
npm install
# 或使用 yarn / pnpm
```

### 3. 启动开发服务器
```bash
npm run dev
# 或 yarn dev / pnpm dev
```

### 4. 访问前端
打开浏览器访问: http://localhost:3000

## 启动后端

### 1. 进入后端目录
```bash
cd backend
```

### 2. 启动 Spring Boot 应用
```bash
# 方式一：前台运行（控制台可见日志）
mvn spring-boot:run -DskipTests

# 方式二：后台运行
nohup mvn spring-boot:run -DskipTests > nohup.out 2>&1 &
```

### 3. 验证后端运行
- 健康检查: http://localhost:8083/actuator/health
- Swagger API文档: http://localhost:8083/swagger-ui/index.html
- API根路径: http://localhost:8083/

## 停止服务

### 停止前端
```bash
# 在运行前端的终端中按 Ctrl+C
```

### 停止后端
```bash
# 查找Java进程
ps aux | grep java

# 杀死进程（使用PID）
kill -9 <进程ID>
```

## 重启服务

### 重启后端
```bash
cd backend
# 先停止
pkill -f "java.*shop-system"
# 再启动
nohup mvn spring-boot:run -DskipTests > nohup.out 2>&1 &
```

### 重启前端
```bash
cd frontend
# 先停止 (Ctrl+C)
# 再启动
npm run dev
```

## 依赖服务

### MySQL 数据库
- 端口: 3306
- 数据库: shop_system
- 配置文件: `backend/src/main/resources/application.yml`

### Redis
- 端口: 6379
- 配置文件: `backend/src/main/resources/application.yml`

## 快速启动脚本（可选）

创建 `start.sh`:
```bash
#!/bin/bash
# 启动后端
cd backend
nohup mvn spring-boot:run -DskipTests > ../backend.log 2>&1 &
echo "后端启动中..."

# 等待后端启动
sleep 5

# 启动前端
cd ../frontend
npm run dev &
echo "前端启动中..."

echo "✅ 服务启动完成"
echo "前端: http://localhost:3000"
echo "后端API: http://localhost:8083"
echo "Swagger文档: http://localhost:8083/swagger-ui/index.html"
```

## 故障排除

### 端口冲突
- 8083端口被占用: 修改 `backend/src/main/resources/application.yml` 中的 `server.port`
- 3000端口被占用: 修改 `frontend/vite.config.ts` 中的 `server.port`

### 数据库连接失败
- 确保MySQL服务运行: `sudo systemctl status mysql`
- 检查用户名密码: `backend/src/main/resources/application.yml`

### Redis连接失败
- 确保Redis服务运行: `redis-cli ping`
